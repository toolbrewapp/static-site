01 · What's in the box

Ten tools. One pack.

Each piece is battle-tested against real repos. They share a voice, compose cleanly, and do one thing well.

Command
/commit

Conventional commit messages from your staged diff, in your project's style.

Command
/pr

A full PR description from the branch diff — follows your template if you have one.

Command
/review

Thorough code review with Blocker / Should-fix / Nit priorities. No wall-of-text.

Command
/security

OWASP-aware pass on pending changes. Catches what a tired reviewer misses at 6pm.

Command
/test

Tests for the current file in your project's style — detects jest / vitest / pytest / go test.

Command
/refactor

Three refactor options with tradeoffs, not one opinionated rewrite. Readability vs. testability vs. performance.

Skill
docs-updater

Auto-activates when a public signature changes. Keeps JSDoc, docstrings, and README snippets in sync.

Skill
migration-writer

Writes up and down migrations. Supports Prisma, Drizzle, and Alembic out of the box.

Subagent
test-fixer

Fixes failing tests in an isolated loop. Reports back with a single summary — doesn't pollute the main context.

Hook
secret-scanner

Pre-edit hook that blocks writes introducing AWS, Stripe, OpenAI, Anthropic, or GitHub secrets.
